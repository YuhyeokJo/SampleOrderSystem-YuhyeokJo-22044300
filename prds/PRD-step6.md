# PRD-step6.md - 모니터링 기능 명세

## 개요
담당자가 현재 시스템 상태(주문 현황, 재고 현황)를 한눈에 파악할 수 있도록 두 가지 조회 기능을
제공한다. 이번 step은 순수 조회 기능이며, 시료/주문/생산 데이터를 변경하지 않는다. 콘솔(터미널)
기반으로 담당자가 메뉴를 선택해 조회한다.

## "주문 대비 재고" 해석 (Plan-step6.md 설계 결정)
- 시료의 재고는 승인(Step 4) 시점에 이미 소비되므로, `CONFIRMED`/`PRODUCING`/`REJECTED`/
  `RELEASE` 상태의 주문은 재고 반영이 끝난 주문이다.
- "주문 대비 재고 충분/부족" 판단에 사용하는 "주문"은 아직 처리되지 않은 `RESERVED` 상태의
  주문만을 의미한다. 시료별로 그 시료를 대상으로 하는 RESERVED 주문들의 주문 수량을 모두
  더한 값을 "수요"로 정의한다.

## 기능 명세

### 1. 주문량 확인
- 입력: 없음
- 처리: 저장된 모든 주문을 상태별로 집계한다. 집계 대상 상태는 `RESERVED`, `CONFIRMED`,
  `PRODUCING`, `RELEASE` 4가지이며, `REJECTED` 상태의 주문은 집계에서 제외한다(유효한 주문이
  아닌 것으로 간주).
- 출력: 4가지 상태 각각의 주문 건수를 표시한다. 특정 상태의 주문이 하나도 없으면 해당 상태의
  건수는 0으로 표시한다(전체를 비어 있음으로 안내하지 않는다).

### 2. 재고량 확인
- 입력: 없음
- 처리: 등록된 모든 시료에 대해 다음을 계산한다.
  1. 해당 시료의 현재 재고 수량을 조회한다.
  2. 해당 시료를 대상으로 하는 `RESERVED` 상태 주문들의 주문 수량 합(수요)을 계산한다
     (RESERVED 주문이 없으면 수요는 0).
  3. 재고 상태를 판정한다:
     - 재고 수량이 정확히 0이면 **고갈**(수요와 무관하게 항상 고갈로 표기).
     - 재고 수량이 0보다 크고, 수요 이상이면 **여유**.
     - 재고 수량이 0보다 크고, 수요보다 적으면 **부족**.
- 출력: 각 시료의 ID, 이름, 현재 재고 수량, 재고 상태(여유/부족/고갈)를 표 형태로 출력한다.
  등록된 시료가 없으면 목록이 비어 있음을 안내한다.

## 데이터 조회 방식
- 이번 step은 새로운 저장소를 추가하지 않는다. Step 1의 `SampleModel.all()`과 Step 2~5의
  `OrderModel.all()`/`OrderModel.list_reserved()`를 조회해 그 자리에서 집계만 수행한다.
- 조회 결과를 저장하거나 시료/주문 데이터를 수정하는 동작은 없다.

## 콘솔 입출력 구조
- Step 1~5와 동일하게 `ConsoleMVC-YuhyeokJo-22044300`의 model(데이터 집계)·view(콘솔
  입출력)·controller(흐름 제어) 계층 분리를 따른다.
- `DataMonitor-YuhyeokJo-22044300`처럼 상태를 표 형태로 콘솔에 출력하는 방식을 참고하되,
  실시간 파일 감시(watchdog) 기능은 가져오지 않는다(이번 step은 메뉴 선택 시 1회 조회하는
  방식이다).

## 예외/경계 케이스 (테스트에 반드시 포함)
- 주문이 전혀 없는 상태에서 주문량 확인 → 4가지 상태 모두 0건으로 표시
- REJECTED 상태의 주문이 있어도 주문량 확인 집계에 포함되지 않는지 확인
- 여러 상태의 주문이 섞여 있을 때 각 상태별 건수가 정확히 집계되는지 확인
- 시료가 전혀 없는 상태에서 재고량 확인 → 빈 목록 안내
- 재고 수량이 정확히 0인 시료 → RESERVED 주문 존재 여부와 무관하게 고갈로 표기
- 재고 수량이 0보다 크고 해당 시료의 RESERVED 주문 수량 합과 정확히 같은 경계값 → 여유로 표기
- 재고 수량이 0보다 크고 해당 시료의 RESERVED 주문 수량 합보다 1 적은 경우 → 부족으로 표기
- 재고 수량이 0보다 큰데 해당 시료를 대상으로 하는 RESERVED 주문이 하나도 없는 경우 → 여유로
  표기
- 한 시료에 대해 여러 건의 RESERVED 주문이 있을 때 수량 합산이 정확한지 확인
- CONFIRMED/PRODUCING/REJECTED/RELEASE 상태의 주문 수량은 재고 상태 판정의 수요 계산에
  포함되지 않는지 확인

## 범위 제외
- 시료/주문/생산/승인·거절/출고 데이터를 변경하는 기능은 정의하지 않는다(조회 전용).
- 메인 메뉴 통합은 이 문서의 범위가 아니다(Step 7에서 다룬다).
