# PRD-step1.md - 시료 관리 기능 명세

## 개요
시스템에 등록된 시료만 이후 주문 대상이 될 수 있으므로, 시료 데이터를 등록·조회·검색하는
기능을 정의한다. 콘솔(터미널) 기반으로 담당자가 직접 명령을 입력하여 사용한다.

## 도메인 모델

| 필드 | 타입 | 제약 |
|---|---|---|
| 시료 ID | 문자열 또는 정수(고유값) | 시스템 내 중복 불가 |
| 이름 | 문자열 | 비어있지 않음 |
| 평균 생산시간 | 숫자 (min/ea) | 0보다 커야 함 (`> 0`) |
| 수율 | 소수 | `0.0 <= yield_rate <= 1.0` |
| 재고 수량 | 정수 | `0` 이상 |

## 기능 명세

### 1. 시료 등록
- 입력: 시료 ID, 이름, 평균 생산시간, 수율, 초기 재고 수량 (재고 수량은 사용자가 직접 입력)
- 처리:
  - 위 도메인 모델 제약을 모두 만족하는지 검증한다.
  - 이미 존재하는 시료 ID로 등록을 시도하면 오류를 안내하고 등록을 거부한다.
  - 제약을 만족하지 않는 입력(평균 생산시간 ≤ 0, 수율 범위 초과, 재고 수량 음수/정수 아님)은
    오류를 안내하고 등록을 거부한다.
  - 모든 검증을 통과하면 시료를 저장하고 등록 완료를 안내한다.
- 출력: 등록 성공/실패 결과 메시지

### 2. 시료 목록 조회
- 입력: 없음
- 처리: 등록된 모든 시료를 조회한다.
- 출력: 각 시료의 ID, 이름, 평균 생산시간, 수율, 현재 재고 수량을 표 형태로 출력한다.
  등록된 시료가 없으면 목록이 비어 있음을 안내한다.

### 3. 시료 검색
- 입력: 검색어(이름 기준)
- 처리: 입력한 검색어가 이름에 포함된 시료를 조회한다.
- 출력: 일치하는 시료 목록(목록 조회와 동일한 항목 구성)을 출력한다. 일치하는 시료가 없으면
  없음을 안내한다.

## 저장 방식
- 프로세스를 재시작해도 등록된 시료 데이터가 유지되도록 파일(JSON) 기반으로 영속화한다.
- 참고: `DataPersistence-YuhyeokJo-22044300`의 임시 파일 작성 후 원자적 교체(`os.replace`) 방식.

## 콘솔 입출력 구조
- 참고: `ConsoleMVC-YuhyeokJo-22044300`의 model(데이터/검증)·view(입출력)·controller(흐름 제어)
  계층 분리를 따른다.
- model은 콘솔 입출력을 포함하지 않고, view는 비즈니스 로직을 포함하지 않는다.

## 예외/경계 케이스 (테스트에 반드시 포함)
- 중복된 시료 ID로 등록 시도 → 등록 거부
- 평균 생산시간이 0 또는 음수 → 등록 거부
- 수율이 0.0 미만 또는 1.0 초과 → 등록 거부
- 수율이 정확히 0.0 또는 1.0인 경계값 → 등록 허용
- 재고 수량이 음수이거나 정수가 아님 → 등록 거부
- 재고 수량이 정확히 0인 경계값 → 등록 허용
- 등록된 시료가 없는 상태에서 목록 조회/검색 → 빈 결과 안내
- 검색어와 일치하는 시료가 없는 경우 → 없음 안내
- 프로세스를 재시작한 뒤(새 인스턴스로) 목록 조회 시 이전에 등록한 시료가 그대로 남아 있는지 확인

## 범위 제외
- 시료 수정, 삭제 기능은 정의하지 않는다(요구사항에 명시되지 않음).
- 주문, 생산, 승인/거절, 출고, 모니터링 관련 기능은 이 문서의 범위가 아니다.
