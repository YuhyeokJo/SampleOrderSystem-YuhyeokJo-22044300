# PRD-step2.md - 시료 주문 접수 기능 명세

## 개요
고객이 이메일 등으로 시료를 요청하면, 주문 담당자가 해당 요청에 맞는 주문서를 시스템에
접수한다. 접수된 주문은 이후 승인/거절 단계(Step 4)에서 처리 대상이 된다. 콘솔(터미널) 기반으로
담당자가 직접 명령을 입력하여 사용한다.

## 도메인 모델

| 필드 | 타입 | 제약 |
|---|---|---|
| 주문번호 | 문자열 | 시스템이 자동 생성, 고유값, 형식 `ORD-YYYYMMDD-XXXX` |
| 시료 ID | 문자열 | Step 1에 등록된 시료 중 하나를 참조해야 함(존재하지 않으면 주문 불가) |
| 고객명 | 문자열 | 비어있지 않음 |
| 주문 수량 | 정수 | `1` 이상 |
| 상태 | 문자열 | 접수 시점에는 항상 `RESERVED` |

## 주문번호 생성 규칙
- 형식: `ORD-YYYYMMDD-XXXX`
  - `YYYYMMDD`: 주문 접수 시점의 날짜(연-월-일, 8자리)
  - `XXXX`: 같은 날짜 내에서 접수된 순서를 나타내는 4자리 일련번호(`0001`부터 시작, 1씩 증가).
    같은 날짜에 이미 3자리로 표현 불가능할 만큼(9999건 초과) 주문이 쌓이는 경우는 이번 step의
    고려 대상이 아니다.
  - 예: 그 날의 첫 주문이 2026-07-15에 접수되면 `ORD-20260715-0001`
- 날짜가 바뀌면 일련번호는 다시 `0001`부터 시작한다.
- 생성된 주문번호는 시스템 내에서 고유해야 한다.

## 기능 명세

### 1. 시료 예약 (주문 접수)
- 입력: 시료 ID, 고객명, 주문 수량
- 처리:
  - 입력한 시료 ID가 Step 1에 등록된 시료 중에 존재하는지 확인한다. 존재하지 않으면 오류를
    안내하고 접수를 거부한다.
  - 고객명이 비어있지 않은지, 주문 수량이 1 이상의 정수인지 검증한다. 제약을 만족하지 않으면
    오류를 안내하고 접수를 거부한다.
  - 모든 검증을 통과하면 주문번호를 자동 생성하고, 상태를 `RESERVED`로 설정하여 주문을
    저장한다.
- 출력: 접수 성공 시 생성된 주문번호를 포함한 성공 메시지, 실패 시 실패 사유 메시지

### 2. 접수된 주문 목록 조회
- 입력: 없음
- 처리: 저장된 모든 주문을 조회한다.
- 출력: 각 주문의 주문번호, 시료 ID, 고객명, 주문 수량, 상태를 표 형태로 출력한다. 등록된
  주문이 없으면 목록이 비어 있음을 안내한다.

## 저장 방식
- 프로세스를 재시작해도 접수된 주문 데이터가 유지되도록 파일(JSON) 기반으로 영속화한다.
- 참고: `DataPersistence-YuhyeokJo-22044300`의 임시 파일 작성 후 원자적 교체(`os.replace`) 방식,
  Step 1의 시료 저장 방식과 동일한 패턴을 적용한다.

## 콘솔 입출력 구조
- Step 1과 동일하게 `ConsoleMVC-YuhyeokJo-22044300`의 model(데이터/검증)·view(콘솔 입출력)·
  controller(흐름 제어) 계층 분리를 따른다.
- 주문 접수 시 시료 존재 여부 확인은 Step 1의 시료 조회 기능(`SampleModel`)을 재사용한다.

## 예외/경계 케이스 (테스트에 반드시 포함)
- 등록되지 않은 시료 ID로 주문 시도 → 접수 거부
- 고객명이 빈 문자열 → 접수 거부
- 주문 수량이 0 또는 음수 → 접수 거부
- 주문 수량이 정수가 아닌 값 → 접수 거부
- 주문 수량이 정확히 1인 경계값 → 접수 허용
- 같은 날짜에 여러 건 접수 시 주문번호의 일련번호가 `0001`부터 순서대로 증가하는지 확인
- 접수된 주문이 없는 상태에서 목록 조회 → 빈 결과 안내
- 프로세스를 재시작한 뒤(새 인스턴스로) 목록 조회 시 이전에 접수한 주문이 상태값(`RESERVED`)까지
  그대로 남아 있는지 확인

## 범위 제외
- 주문 승인/거절(재고 판단에 따른 CONFIRMED/PRODUCING/REJECTED 전환)은 Step 4에서 다룬다.
- 생산라인, 출고 처리, 모니터링, 메인 메뉴 통합은 이 문서의 범위가 아니다.
- 주문 수정, 취소 기능은 정의하지 않는다(요구사항에 명시되지 않음).
