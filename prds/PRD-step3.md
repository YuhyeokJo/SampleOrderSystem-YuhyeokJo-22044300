# PRD-step3.md - 생산라인 기능 명세

## 개요
주문 승인 시 재고가 부족하면 부족분을 생산 라인에 등록해 생산한다. 하나의 생산 라인은 시료를
하나씩만 생산하며, 주문이 들어온 시료에 대해서만 생산한다. 생산이 완료되면 해당 주문은 출고
대기 상태(CONFIRMED)로 전환된다. 콘솔(터미널) 기반으로 담당자가 직접 명령을 입력하여 생산
진행을 기록한다.

## 도메인 모델

### 생산 항목 (ProductionJob)

| 필드 | 타입 | 설명/제약 |
|---|---|---|
| 주문번호 | 문자열 | 대상 주문의 주문번호(Step 2의 `Order.order_id`) |
| 시료 ID | 문자열 | 대상 시료 ID(Step 1의 `Sample.sample_id`) |
| 부족분 | 정수 | 생산이 필요한 수량, `1` 이상 |
| 실 생산량 | 정수 | `ceil(부족분 / (수율 × 0.9))`, 등록 시점에 계산되어 고정 |
| 총 생산 시간 | 숫자 | `평균 생산시간 × 실 생산량`, 등록 시점에 계산되어 고정 |
| 누적 생산량 | 정수 | 초기값 0, 생산 진행 기록마다 누적, `실 생산량`을 초과할 수 없음 |
| 등록 순서 | 정수 또는 타임스탬프 | FIFO 순서 판단용, 등록 시 자동 부여 |

### 주문 상태 갱신 (`OrderModel` 확장)
- Step 2의 `OrderModel`에 주문 상태를 변경하고 재저장하는 기능을 추가한다
  (예: `update_status(order_id, new_status)`).
- 존재하지 않는 주문번호로 상태 변경을 시도하면 오류를 안내하고 거부한다.
- 이 기능 자체는 상태값의 종류를 제한하지 않는다(`RESERVED`/`PRODUCING`/`CONFIRMED`/`REJECTED`/
  `RELEASE` 중 어떤 값으로도 전환 가능). 상태 전환 규칙(어떤 상태에서 어떤 상태로만 전환 가능한지)의
  검증은 이를 호출하는 쪽(이번 step의 생산 완료 처리, 이후 Step 4의 승인/거절)의 책임으로 한다.

## 생산량 계산 규칙 (등록 시 1회 계산)
- 실 생산량 = `ceil(부족분 / (수율 × 0.9))` (정수로 올림)
- 총 생산 시간 = `평균 생산시간 × 실 생산량`
- 계산에 필요한 수율/평균 생산시간은 생산 항목의 시료 ID로 Step 1의 `SampleModel`을 조회하여
  가져온다. 조회되지 않는 시료 ID로 등록을 시도하면 오류를 안내하고 거부한다.

## 기능 명세

### 1. 생산 큐 등록
- 입력: 주문번호, 시료 ID, 부족분
- 처리:
  - 시료 ID가 Step 1에 존재하는지 확인, 없으면 등록 거부
  - 부족분이 1 이상의 정수인지 확인, 아니면 등록 거부
  - 실 생산량/총 생산 시간을 계산하여 생산 항목을 생성하고, 누적 생산량 0으로 큐의 맨 뒤에
    추가한다(FIFO).
- 출력: 등록 성공 시 계산된 실 생산량/총 생산 시간을 포함한 안내, 실패 시 사유 안내
- 참고: 이 기능은 이후 Step 4의 주문 승인 로직이 재고 부족 시 호출할 공개 API이며, 이번
  step에서는 이 등록 함수를 직접 호출하는 방식으로 검증한다. 등록 시점에 대상 주문의 상태를
  검증하지는 않는다(주문 상태 검증은 Step 4의 책임).

### 2. 생산 진행 기록
- 입력: 이번에 생산된 수량(양의 정수)
- 처리:
  - 현재 진행 중인 생산 항목(큐의 맨 앞, 없으면 오류 안내 후 거부)에 대해 누적 생산량을 더한다.
  - 더한 값이 실 생산량 이상이 되면(초과분은 버리고 실 생산량으로 맞춘다) 생산 완료로 판단하고,
    아래 "3. 생산 완료 처리"를 수행한다.
  - 실 생산량 미만이면 누적 생산량만 갱신하고 종료한다.
- 출력: 갱신된 누적 생산량/실 생산량 안내, 완료된 경우 완료 안내

### 3. 생산 완료 처리
- 처리:
  - 대상 주문의 상태를 `PRODUCING`에서 `CONFIRMED`로 변경한다(`OrderModel.update_status` 사용).
  - 완료된 생산 항목을 큐에서 제거한다.
  - 큐에 다음 대기 항목이 있으면 그 항목이 새로운 진행 중 항목이 된다(누적 생산량은 0에서 시작).
- 출력: 완료된 주문번호와 전환된 상태(CONFIRMED) 안내

### 4. 생산 현황 조회
- 입력: 없음
- 처리: 현재 진행 중인 생산 항목(큐의 맨 앞)을 조회한다.
- 출력: 주문번호, 시료 ID, 부족분, 실 생산량, 총 생산 시간, 누적 생산량을 표시한다. 진행 중인
  항목이 없으면(큐가 비어 있으면) 없음을 안내한다.

### 5. 대기 주문 확인
- 입력: 없음
- 처리: 큐에서 현재 진행 중인 항목을 제외한 나머지 대기 항목을 FIFO 순서(등록 순서)대로 조회한다.
- 출력: 각 대기 항목의 주문번호, 시료 ID, 부족분, 실 생산량, 총 생산 시간을 표 형태로 출력한다.
  대기 항목이 없으면 없음을 안내한다.

  영속화한다.
- 참고: `DataPersistence-YuhyeokJo-22044300`의 임시 파일 작성 후 원자적 교체(`os.replace`) 방식,
  Step 1/Step 2와 동일한 패턴을 적용한다.

## 콘솔 입출력 구조
- Step 1/Step 2와 동일하게 `ConsoleMVC-YuhyeokJo-22044300`의 model(데이터/검증)·view(콘솔
  입출력)·controller(흐름 제어) 계층 분리를 따른다.

## 예외/경계 케이스 (테스트에 반드시 포함)
- 등록되지 않은 시료 ID로 생산 큐 등록 시도 → 등록 거부
- 부족분이 0 또는 음수, 정수가 아닌 값 → 등록 거부
- 실 생산량 계산 시 나눗셈 결과가 정수가 아닌 경우 올림 처리가 정확한지 확인
  (예: 부족분 10, 수율 0.9 → `10 / (0.9*0.9) = 12.34...` → 실 생산량 13)
- 진행 중인 항목이 없는 상태에서 생산 진행 기록 시도 → 거부
- 생산 진행 기록을 여러 번에 나누어 입력했을 때 누적 생산량이 올바르게 합산되는지 확인
- 누적 생산량이 실 생산량에 정확히 도달했을 때 자동으로 완료 처리되고 주문 상태가 CONFIRMED로
  바뀌는지 확인
- 한 번에 실 생산량을 초과하는 수량을 입력해도 누적 생산량이 실 생산량을 넘지 않고 완료 처리되는지
  확인
- 완료 처리 후 큐에 다음 대기 항목이 있으면 그 항목이 새로운 진행 중 항목이 되는지 확인
- 큐가 비어 있는 상태에서 생산 현황 조회/대기 주문 확인 → 각각 없음 안내
- 여러 항목을 등록했을 때 대기 주문 확인이 FIFO(등록 순서)를 지키는지 확인
- 프로세스를 재시작한 뒤(새 인스턴스로) 생산 큐를 다시 읽었을 때 등록된 항목과 누적 생산량이
  그대로 남아 있는지 확인
- 존재하지 않는 주문번호로 `OrderModel.update_status` 호출 시 거부되는지 확인

## 범위 제외
- 주문 승인/거절 로직(재고 판단에 따른 RESERVED → CONFIRMED/PRODUCING/REJECTED 최초 전환)은
  Step 4에서 다룬다.
- 출고 처리, 모니터링, 메인 메뉴 통합은 이 문서의 범위가 아니다.
- 여러 생산 라인을 두어 병렬로 생산하는 기능은 정의하지 않는다(하나의 생산 라인은 시료를
  하나씩만 생산).
- 생산 진행 기록의 자동화(예: 시간 경과에 따른 자동 생산량 증가)는 정의하지 않는다. 생산량 기록은
  담당자의 수동 입력으로만 이루어진다.
