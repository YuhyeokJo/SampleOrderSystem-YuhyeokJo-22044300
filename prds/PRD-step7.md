# PRD-step7.md - 메인 메뉴 통합 기능 명세

## 개요
지금까지 구현한 6개 기능(시료 관리/시료 주문/주문 승인·거절/모니터링/생산라인/출고 처리)을
하나의 메인 메뉴에서 선택해 실행할 수 있도록 통합한다. 메뉴 화면에는 전체 시료에 대한 요약
정보를 함께 표시한다. 콘솔(터미널) 기반으로 담당자가 메뉴 번호를 입력해 사용한다.

## 시료 요약 정보 (Plan-step7.md 설계 결정)
- 표시 항목: **등록된 시료 종류 수**(예: "등록된 시료: 3종"), **전체 시료의 재고 수량 합계**
  (예: "총 재고 수량: 128").
- 계산 방법: `SampleModel.all()`로 전체 시료를 조회하여 개수를 세고, 각 시료의
  `stock_quantity`를 모두 더한다.
- 등록된 시료가 하나도 없으면 "등록된 시료가 없습니다."로 표시하고, 재고 합계는 표시하지
  않는다.
- 요약 정보는 메인 메뉴가 표시될 때마다(최초 진입 시, 그리고 하위 컨트롤러 종료 후 메인
  메뉴로 복귀할 때 모두) 새로운 `SampleModel` 인스턴스를 생성해 다시 조회한 값으로 계산한다.
  이전에 계산한 값을 캐시하여 재사용하지 않는다. 이는 하위 메뉴(예: 시료 관리)에서 시료를
  새로 등록하고 돌아왔을 때 요약 정보가 최신 상태로 갱신되어야 하기 때문이다.

## 메인 메뉴 구성
| 번호 | 항목 | 실행 대상 |
|---|---|---|
| 1 | 시료 관리 | `SampleController` |
| 2 | 시료 주문 | `OrderController` |
| 3 | 주문 승인/거절 | `ApprovalController` |
| 4 | 모니터링 | `MonitoringController` |
| 5 | 생산라인 | `ProductionController` |
| 6 | 출고 처리 | `ReleaseController` |
| 7 | 프로그램 종료 | 프로그램 종료 |

## 기능 명세

### 1. 메인 메뉴 표시 및 선택 처리
- 처리:
  1. 시료 요약 정보를 조회해 화면 상단에 표시한다.
  2. 위 표(1~7번)의 메뉴 항목을 표시한다.
  3. 사용자의 입력(번호)을 받는다.
  4. 1~6번을 선택하면 대응하는 컨트롤러의 `run()`을 호출한다. 해당 컨트롤러는 각자 자체
     메뉴 루프를 가지고 있으며, 그 컨트롤러 내에서 담당자가 종료를 선택하면 제어가 메인
     메뉴로 되돌아온다(메인 메뉴가 다시 요약 정보와 함께 표시된다).
  5. 7번을 선택하면 프로그램을 종료한다(종료 안내 메시지 출력 후 메인 메뉴 루프를 빠져나감).
  6. 1~7 범위를 벗어나거나 숫자가 아닌 입력이면 오류를 안내하고 메인 메뉴를 다시 표시한다.
- 각 하위 컨트롤러(`SampleController` 등)의 내부 메뉴 항목, 유효성 검증, 상태 전환 로직은
  Step 1~6에서 이미 구현되어 있으므로 이 문서에서 다시 정의하지 않으며, 이번 step에서 해당
  코드를 수정하지 않는다.

## `main.py` 연결
- `main.py`는 위 메인 메뉴 컨트롤러를 생성해 실행하는 것으로 변경한다(기존에 `SampleController`
  만 직접 실행하던 부분을 대체한다).

## 콘솔 입출력 구조
- Step 1~6과 동일하게 `ConsoleMVC-YuhyeokJo-22044300`의 model(데이터 집계)·view(콘솔
  입출력)·controller(흐름 제어) 계층 분리를 따른다. 이번 step은 별도의 도메인 model이
  필요하지 않으면 view/controller만으로 구성해도 무방하다(요약 정보 계산은 기존
  `SampleModel`을 그대로 사용하므로 신규 model 클래스가 반드시 필요하지는 않다).

## 예외/경계 케이스 (테스트에 반드시 포함)
- 등록된 시료가 없는 상태에서 메인 메뉴를 표시하면 "등록된 시료가 없습니다."로 표시되는지 확인
- 시료가 여러 개 등록되어 있을 때 종류 수와 재고 합계가 정확히 계산되는지 확인
- 1~6번 각 선택 시 대응하는 컨트롤러의 `run()`이 정확히 호출되는지 확인
- 7번 선택 시 프로그램(메인 메뉴 루프)이 정상적으로 종료되는지 확인
- 범위를 벗어난 숫자(예: 0, 8)나 숫자가 아닌 입력 시 오류 안내 후 메뉴가 다시 표시되는지 확인
- 하위 컨트롤러 실행이 끝난 뒤 메인 메뉴로 정상적으로 복귀하는지 확인
- 시료 관리(1번) 실행 중 새로운 시료를 등록하고 메인 메뉴로 돌아왔을 때, 요약 정보(종류 수/
  재고 합계)가 캐시된 이전 값이 아니라 갱신된 값으로 표시되는지 확인

## 범위 제외
- Step 1~6에서 구현한 각 기능의 내부 동작(등록, 주문 접수, 승인/거절, 모니터링 집계, 생산
  큐 처리, 출고 처리)은 이 문서의 범위가 아니며 수정하지 않는다.
- 새로운 도메인 기능(재고/주문/생산/승인/출고 관련 새로운 동작)은 추가하지 않는다.
